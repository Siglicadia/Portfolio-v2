Headings

Markdown
# Heading 1 ## Heading 2 ### Heading 3 #### Heading 4 ##### Heading 5 ###### Heading 6
Rendered

Heading 1

Heading 2

Heading 3

Heading 4

Heading 5

Heading 6

Gotcha

You must have a space after the # — #Heading does not render as a heading in most parsers.

Emphasis

Markdown
**bold text** __also bold__ *italic text* _also italic_ ***bold and italic*** ~~strikethrough~~
Rendered

bold text

also bold

italic text

also italic

bold and italic

strikethrough

Tip

Prefer ** for bold and * for italic — underscores can cause issues mid-word (e.g. some_var_name may render unexpectedly).

Paragraphs & line breaks

What you wantHow to do it
New paragraphLeave a blank line between text
Line break (soft)End a line with two spaces, then press Enter
Hard line breakUse <br> (HTML, works in most flavours)
Horizontal rule--- or *** or ___ (three or more)
Gotcha

A single line break in your source is ignored. You need a blank line for a new paragraph, or two trailing spaces for a line break within a paragraph.

Unordered lists

Markdown
- Item one - Item two - Nested item - Another nested - Item three * Also works with asterisks + Or plus signs
Rendered
  • Item one
  • Item two
    • Nested item
    • Another nested
  • Item three
Tip

Nest items by indenting with 2 or 4 spaces (varies by flavour). Be consistent — mixing indent levels without proper spacing breaks nesting.

Ordered lists

Markdown
1. First item 2. Second item 1. Nested numbered 2. Also nested 3. Third item 1. You can use 1. for all 1. Markdown auto-increments 1. The numbers
Rendered
  1. First item
  2. Second item
    1. Nested numbered
    2. Also nested
  3. Third item
Key fact

The actual numbers you use don't matter — Markdown only uses the first number in a list to set the start, then auto-increments. All 1. is perfectly valid.

Task lists (GFM)

Markdown
- [x] Completed task - [ ] Incomplete task - [x] Another done item
Rendered
  • ✅ Completed task
  • ⬜ Incomplete task
  • ✅ Another done item

Task lists are a GitHub Flavored Markdown (GFM) extension — not in the original spec. Works on GitHub, GitLab, many editors. The space inside [ ] is required for unchecked.

Loose vs tight lists

Tight (no blank lines)
- Item one - Item two - Item three
Loose (blank lines between)
- Item one - Item two - Item three

Loose lists wrap each item in a <p> tag, adding vertical spacing. Tight lists don't. This is a common test question.

Inline links

Markdown
[Link text](https://example.com) [With title](https://example.com "Tooltip text") [Relative link](./other-page.md) <https://auto-linked.com> <email@example.com>
Rendered

Link text

With title

Relative link

https://auto-linked.com

email@example.com

Reference-style links

[Link text][ref-id] [Another link][1] <!-- Definitions anywhere in the document: --> [ref-id]: https://example.com "Optional title" [1]: https://another.com

Definitions can go anywhere — top, bottom, or inline. IDs are case-insensitive. Useful for reusing the same URL multiple times.

Images

Markdown
![Alt text](image.png) ![Alt](image.png "Optional title") <!-- Reference style: --> ![Alt text][img-ref] [img-ref]: image.png "Title"
Notes

Same syntax as links but with ! prefix.

Alt text is important for accessibility and appears if image fails to load.

Markdown alone can't set image width/height — use HTML <img> for that.

Linking an image

<!-- Wrap image syntax in link syntax: --> [![Alt text](image.png)](https://example.com) <!-- Image reference inside link: --> [![Alt][img-id]][link-id]

Images and links can be combined by wrapping the ![...](...) syntax inside a [...](...) link.

Inline code

Markdown
Use `backticks` for inline code. Use ``double backticks if your code contains a `backtick` itself``.
Rendered

Use backticks for inline code.

Use double backticks if your code contains a `backtick` itself.

Fenced code blocks

```python def hello(name): print(f"Hello, {name}!") ``` ~~~javascript const greet = name => `Hello, ${name}!`; ~~~
Tip

Specify a language after the opening fence for syntax highlighting. Both backtick fences (```) and tilde fences (~~~) work. The language identifier is optional but recommended.

Indented code blocks (original spec)

Normal paragraph here. This is a code block. Indented with 4 spaces or 1 tab. No language highlighting support. Back to normal.
Gotcha

Indented code blocks are in the original Markdown spec but fenced blocks (```) are preferred in modern flavours. A blank line before the indented block is required.

Basic table syntax (GFM)

Markdown
| Name | Score | Grade | |-------|-------|-------| | Alice | 95 | A | | Bob | 82 | B | | Carol | 71 | C |
Rendered
NameScoreGrade
Alice95A
Bob82B
Carol71C

Tables are a GFM extension — not in the original Markdown spec. The separator row (dashes) is required. Column widths don't need to align visually, but it helps readability.

Column alignment

Markdown
| Left | Centre | Right | |:------|:------:|------:| | text | text | text | | more | more | more |
How colons control alignment

|:---| — left aligned (default)

|:---:| — centre aligned

|---:| — right aligned

Table tips & limits

TopicDetail
Outer pipesOptional but recommended for clarity. Name | Score works without leading/trailing |.
Cell contentCan contain inline formatting: bold, italic, inline code, links
Merging cellsNot supported in standard Markdown — use HTML <table> for that
Multi-line cellsNot supported — each row must be one line
Minimum dashesAt least one - per separator cell is required

Escaping special characters

Prefix with \ to render these literally:

\ ` * _ { } [ ] ( ) # + - . ! | \*not italic\* → *not italic* \# not a heading → # not a heading

HTML in Markdown

<!-- Most flavours allow raw HTML --> <strong>Bold via HTML</strong> <br> <details> <summary>Click to expand</summary> Hidden content here. </details> <!-- Useful for things Markdown can't do: --> <img src="image.png" width="300" alt="sized image"> <div align="center">Centred content</div>

Raw HTML works in most Markdown processors (CommonMark, GFM) but may be sanitised on some platforms (e.g. Reddit strips most HTML).

Footnotes (extended syntax)

Markdown

Supported in GFM, Pandoc, MultiMarkdown. Not in CommonMark core spec.

Definitions can go anywhere; they always render at the bottom.

The most commonly tested edge cases

GotchaDetail
Space after ##Heading renders as text in most parsers. Must be # Heading.
Single line break ignoredOne newline = same paragraph. Need blank line for new paragraph, or 2 trailing spaces for a line break.
Underscore in wordssome_var_name — underscores mid-word may not trigger italics in CommonMark, but can in older parsers. Prefer *.
Ordered list numbersOnly the first number matters. 3. item as first item starts list at 3. After that, auto-increments regardless of your numbers.
Blank line before block elementsCode blocks, blockquotes, and lists often need a blank line before them to render correctly, especially after a paragraph.
Indentation in listsTo continue a paragraph inside a list item, indent 4 spaces. To add a code block inside a list, indent 8 spaces.
Table separatorThe |---| row is mandatory. Without it, nothing renders as a table.
Empty task list[ ] must have a space inside. [] renders as literal text, not a checkbox.
Backslash escapesOnly work on ASCII punctuation. \n is not a newline in Markdown — it's literally \n.
ATX vs setext headingsSetext: underline with === or ---. Only H1 and H2. A line of dashes directly under text can accidentally become a heading.

Setext headings (the accidental heading)

Markdown
Heading 1 ========= Heading 2 --------- Some paragraph text -------------------
Explanation

If you put --- on the line directly below any text, it becomes an H2 heading — even if you meant it as a horizontal rule.

Leave a blank line before --- to get a horizontal rule instead.

Trailing spaces — invisible but meaningful

This line ends with two spaces So this becomes a new line within the same paragraph. This line ends with zero spaces So this is joined to the previous line.
Gotcha

Two trailing spaces create a <br>. This is invisible in most editors and is a classic exam trick. Some linters strip trailing spaces automatically, breaking this behaviour.
